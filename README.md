## 1.📝 Descrição

O **FitCare** nasceu para descomplicar o bem-estar de quem vive uma rotina acelerada. O aplicativo centraliza programas de exercícios curtos e eficientes (de 10 a 30 minutos), divididos por modalidades como Yoga, Pilates, Musculação e HIIT. Através de filtros intuitivos, o usuário escolhe a atividade ideal para o seu momento, garantindo que a sua evolução aconteça onde estiver, sem complicações e sem a necessidade de equipamentos complexos.

## 3. ✨Recursos
- 📝 **Gerenciamento de Atividades:** Cadastro, consulta, atualização e remoção de exercícios.
- 🔍 **Sistema de Buscas e Filtros Dinâmicos:** Filtro por Modalidade (yoga, pilates, musculação, hiit).
- 🧮 **Sistema de Cálculo de IMC:** Cálculo do Índice de Massa Corporal (IMC) com base no peso e altura, facilitando o acompanhamento da evolução física e da saúde do usuário.
- 🛡️ **Validação de Dados:** Garantia de integridade com `class-validator` e `class-transformer`.

## 6. 📋 Pré-requisitos

Antes de iniciar, certifique-se de ter as seguintes ferramentas instaladas:

- [Node.js](https://nodejs.org/) (v16+)
- API NestJS ([Repositório da API do FitCare - Backend](https://github.com/Javascript13-Grupo02/Projeto-integrador-02-FitCare))

---

## 7. 🚀 Configuração e Execução

1. Clone o repositório do Projeto
2. Instale as dependências: `npm install`
3. Clone o repositório do Projeto Backend: [Link](https://github.com/Javascript13-Grupo02/Projeto-integrador-02-FitCare)
4. Siga as instruções de **Configuração e Execução** descritas no README do Projeto Backend
5. Adicione o endereço de execução do projeto na variável de ambiente **VITE_API_URL**, no projeto React.
6. Execute o Projeto React: `npm run dev`
7. A aplicação React estará disponível no endereço: `http://localhost:5173`

---

## 8. 📁 Estrutura do Projeto

A arquitetura do front-end foi organizada visando a separação de responsabilidades, facilitando a escalabilidade e a manutenção do código. Abaixo está a estrutura de diretórios principal localizada na pasta `src/`:

```text
src/
|
├── assets/       # Arquivos estáticos, imagens e ícones 
├── components/   # Componentes visuais e independentes 
├── models/       # Interfaces e tipagens de dados do TypeScript
├── pages/        # Telas completas conectadas ao sistema de rotas 
├── services/     # Configuração do Axios e funções de integração com a API Rest
├── App.css       # Estilos globais adicionais da aplicação
├── App.tsx       # Componente raiz, responsável por gerenciar as Rotas de navegação
├── index.css     # Inicialização do Tailwind CSS e definições globais de estilo
└── main.tsx      # Ponto de entrada (Entry Point) que renderiza a aplicação no navegador
